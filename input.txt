ТЕКСТОВОЕ ПОЯСНЕНИЕ АЛГОРИТМА
===============
Алгоритм работает следующим образом

Мы высчитываем сколько максимально может быть использовано технических единиц (комбинация оборудования и 
техники. Это высчитывается путем нахождения суммы сначала всего количества техники различных типов
которые могут быть использованы для этой операции (actt) затем таким же образом сумму всей техники всех
типов, которые могут быть задействованы для этой операции (acto). 

K = min(actt, acto) поскольку если чего-то будет меньше, то того, чего будет больше мы просто не сможем
задействовать

N = Nt * No - (незадейстованные комбинации (нулевое значение в матрицах для элемента с координатами i и j)
Множество всех таких пар <Ti, Oj>, Ti - тип оборудования i; Oi - тип оборудования Oj; которые используются
в данной операции будем называть Combs

Сложность алгоритма составляет О(A(k,n)). Мы высчитываем все пригодные сочетания. Условием пригодности
является наличие необходимого количества техники для данного размещения с повторением.

Далее мы составляем для каждого такого t, где 1 <= t <= k высчитываем множество всех сочетаний с повторениями
A из |Combs| по t, и для каждого такого сочетния составляем уравнение

P1*x1 + P2*x2 + ... + Pt*xt -> min
C1*x1 + C2*x2 + ... + Ct*xt = S
x1 <= H1
x2 <= H2
...
xk <= Ht

С помощью симпликс-метода решаем данное уравнение и получаем нужный нам результат из выходных данных
Или получаем негативный ответ о том, что распределить оборудование при таких условиях невозможно.
